  Task 2: Smart Contracts — LeaderRegistry + CopyGuardBondVault + ReportPayment + RiskOracleAdapter

  Goal: 实现 4 个核心合约，覆盖 bond 全生命周期、leader 注册、付费报告、oracle 权限控制。

  Files:
  - contracts/src/LeaderRegistry.sol
  - contracts/src/CopyGuardBondVault.sol
  - contracts/src/ReportPayment.sol
  - contracts/src/RiskOracleAdapter.sol

  Implementation Notes:
  - LeaderRegistry: owner 可 registerLeader(id, venue, wallet, metadataHash) 和 deactivateLeader(id)。P0 leader 由部署者预注册，不需要用户自由注册。
  - CopyGuardBondVault: 核心合约。createBond(leaderId, amount, riskThresholdBps, expiry) — follower 从钱包直接调用，USDC 计价。Bond 状态机：Active → Warned →
  Slashed/Refunded/Settled。submitRiskUpdate(bondId, riskScoreBps, reportHash) 只允许 RiskOracleAdapter 调用。当 riskScoreBps > riskThresholdBps 时自动触发
  Warned。slashBond、refundBond、settleBond 受权限控制。
  - ReportPayment: purchaseReport(leaderId) 支付 1 USDC，emit ReportPurchased(user, leaderId, amount)。hasPurchased(user, leaderId) 供后端查询。P0 价格硬编码 1
   USDC。
  - RiskOracleAdapter: owner 注册 authorized oracle 地址（P0 即 FastAPI 后端的钱包地址）。submitRiskUpdate 转发给 BondVault。

  Acceptance Criteria:
  - 所有合约 forge build 无错误
  - 合约间调用关系正确：RiskOracleAdapter → BondVault，ReportPayment 独立
  - Bond 完整生命周期可走通：create → risk update → warn → settle/refund

  Test/Build: cd contracts && forge build
